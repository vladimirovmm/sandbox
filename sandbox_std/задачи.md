
## 3. Графовые алгоритмы

Работают с сетями и связями между объектами (графами). Применяются в навигации, социальных сетях, маршрутизации.

**Важные примеры:**
* **Алгоритм Дейкстры (Dijkstra's Algorithm)** — находит кратчайший путь от одной вершины ко всем остальным во взвешенном графе. Сложность: $O((V + E) \log V)$, где $V$ — вершины, $E$ — рёбра.
* **Алгоритм A* (A‑Star)** — улучшенная версия Дейкстры с эвристикой. Используется в играх и GPS‑навигации.
* **Алгоритм Флойда‑Уоршелла (Floyd‑Warshall)** — находит кратчайшие пути между всеми парами вершин. Сложность: $O(V^{3})$.
* **Алгоритм Краскала (Kruskal's Algorithm)** — строит минимальное остовное дерево графа. Используется в сетевых технологиях.

## 4. Динамическое программирование

Решает сложные задачи через разбиение на подзадачи и сохранение промежуточных результатов.

**Типичные задачи:**
* **Задача о рюкзаке (Knapsack Problem)** — выбирает предметы с максимальной ценностью при ограничении по весу.
* **Наибольшая общая подпоследовательность (LCS)** — находит самую длинную последовательность, общую для двух строк. Применяется в сравнении текстов и биоинформатике.
* **Числа Фибоначчи** — классический пример, где рекурсивный подход без кэширования даёт сложность $O(2^{n})$, а с динамическим программированием — $O(n)$.

## 5. Жадные алгоритмы

Делают локально оптимальный выбор на каждом шаге в надежде получить глобальное решение.

**Примеры:**
* **Кодирование Хаффмана** — строит оптимальный префиксный код для сжатия данных.
* **Дробный рюкзак** — вариант задачи о рюкзаке, где предметы можно делить.
* **Алгоритм Прима** — строит минимальное остовное дерево, добавляя на каждом шаге самое дешёвое ребро.

## 6. Хеширование и структуры данных

Оптимизируют доступ к данным через преобразование ключей в индексы.

* **Хеш‑таблицы (Hash Tables)** — обеспечивают средний случай $O(1)$ для вставки и поиска. Используются в словарях, кешах.
* **Деревья поиска (BST, AVL, Red‑Black)** — поддерживают отсортированные данные с логарифмической сложностью операций.

## 7. Криптографические алгоритмы

Защищают данные через шифрование и хеширование.

* **RSA** — асимметричное шифрование на основе сложности факторизации больших чисел.
* **SHA‑256** — алгоритм безопасного хеширования для цифровых подписей и блокчейна.

## 8. Алгоритмы сжатия данных

Уменьшают объём информации без потерь или с потерями.

* **ZIP (Deflate)** — сжатие без потерь для архивов.
* **JPEG** — сжатие изображений с потерями.
* **MP3** — сжатие аудио с потерями.
